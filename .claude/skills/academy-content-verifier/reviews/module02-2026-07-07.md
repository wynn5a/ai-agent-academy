## Findings (most severe first)

### [Minor] Lab 02's skeleton TODO repeats the same "nudge once" imprecision fixed in Lesson 4
- **Where:** `content/modules/module02/lab.ts:69` — `# TODO: end_turn without finish -> nudge once`
- **Standard:** 1. Factual accuracy / 3. Comprehensive depth (consistency with the lesson it assembles)
- **Issue:** This session's Lesson 4 review found that "nudge once" overclaims what the reference implementation does — nothing tracks a one-time nudge, so the branch actually fires every time the model talks without calling `finish`, bounded only by the budget guard. Lesson 4's code and comments were corrected to say so. Lab 02's skeleton — which explicitly instructs students to assemble `Budget`, `SafeExecutor`, `Tracer`, and this same nudge pattern "straight from lessons 4–5" — still carries the old "nudge once" phrasing in its TODO comment, so a student following the lab literally would recreate the same misleading framing that was just fixed at the source.
- **Why it matters:** Small, but it's the kind of inconsistency that undermines a fix — a learner who read the corrected Lesson 4 and then opens the lab skeleton sees language that contradicts what they just learned.
- **Suggested fix:** Update the TODO comment to match, e.g. `# TODO: end_turn without finish -> nudge (repeats until budget trips)`.

### [Polish] `resources.ts`'s OpenAI link points to an old host that now 301-redirects
- **Where:** `content/modules/module02/resources.ts:13` — `https://platform.openai.com/docs/guides/agents`
- **Standard:** 1. Factual accuracy (link freshness)
- **Issue:** This URL 301-redirects to `https://developers.openai.com/api/docs/guides/agents` (confirmed live). The redirect works, so the link isn't broken, but the destination page is now titled "Agents SDK" and reads as an SDK navigation hub rather than the single-narrative "practical guide" the resource's description implies — still topically accurate (guardrails, orchestration, HITL are all there), just a slightly dated framing of what a reader will land on.
- **Why it matters:** Lowest severity — nothing is broken for the reader. Worth a freshen-up next time this file is touched, not urgent on its own.
- **Suggested fix:** Update the URL to `https://developers.openai.com/api/docs/guides/agents` directly (skip the redirect hop).

### [Minor, carried over — not yet fixed] Lesson 2's historical ReAct code block is Anthropic-specific but untagged
- **Where:** `content/modules/module02/lessons/02-react-and-planning.ts:21-57`
- **Standard:** 4. Anthropic + OpenAI compatibility
- **Issue:** Unchanged from the prior review (`reviews/module02-lesson02-2026-07-07.md`): the `react_step` block calls `client.messages.create(..., stop_sequences=[...])` — real Anthropic SDK code — with no `provider: "claude"` tag and no OpenAI `variants` entry, so it silently renders as neutral pseudocode alongside every other lesson-01/03/04/05 code block that correctly tabs this kind of code.
- **Why it matters:** Same as before — a reader could mistake Anthropic-flavored syntax for provider-neutral pseudocode and miss the (teachable) OpenAI-side difference in stop-sequence handling.
- **Suggested fix:** Unchanged from the prior report: tag `provider: "claude"` and add a short OpenAI `variants` entry (`stop=[...]` on Chat Completions, noting the 4-sequence cap from Module 1).
